Next.js は、柔軟性のあるフル stackWeb アプリケーションを作成するための基本構成要素を提供します。**Building Your Application**のガイドでは、これらの機能の使用方法とアプリケーションの動作のカスタマイズ方法について説明しています。

セクションとページは、基本から高度な内容まで順番に整理されており、Next.js アプリケーションを作成する際にそれらを段階的にたどることができます。しかし、任意の順序で読むことも、自分のユースケースに適用するページをスキップして読むこともできます。

<AppOnly>

Next.js が初めての方は、まずは [ルーティング](/ja/javascript/next.js/v14/02-app/01-building-your-application/01-routing)、 [レンダリング](/ja/javascript/next.js/v14/02-app/01-building-your-application/03-rendering)、 [データ取得](/ja/javascript/next.js/v14/02-app/01-building-your-application/02-data-fetching) 、 [スタイリング](/ja/javascript/next.js/v14/02-app/01-building-your-application/05-styling) のセクションから始めることをお勧めします。これらは、あなたが始めるのを助ける基本的な Next.js とウェブの概念を紹介します。その後、 [最適化](/ja/javascript/next.js/v14/02-app/01-building-your-application/06-optimizing) や [設定](/ja/javascript/next.js/v14/02-app/01-building-your-application/07-configuring)などの他のセクションに深く掘り下げることができます。最後に、準備ができたら、 [デプロイ](/ja/javascript/next.js/v14/02-app/01-building-your-application/10-deploying) と [アップグレード](/ja/javascript/next.js/v14/02-app/01-building-your-application/11-upgrading) のセクションを確認してください。

</AppOnly>

<PagesOnly>

あなたが Next.js を初めて使うなら、私たちは最初に[Routing](/ja/javascript/next.js/v14/03-pages/01-building-your-application/01-routing)、[Rendering](/ja/javascript/next.js/v14/03-pages/01-building-your-application/02-rendering)、[Data Fetching](/ja/javascript/next.js/v14/03-pages/01-building-your-application/03-data-fetching)、そして[Styling](/ja/javascript/next.js/v14/03-pages/01-building-your-application/04-styling)のセクションから始めることをお勧めします。これらは、基本的な Next.js と web の概念を紹介して、あなたが始めるのを支援します。その後、他のセクション、例えば[Optimizing](/ja/javascript/next.js/v14/03-pages/01-building-your-application/05-optimizing)と[Configuring](/ja/javascript/next.js/v14/03-pages/01-building-your-application/06-configuring)に深くダイビングすることができます。最後に、あなたが準備ができたら、[Deploying](/ja/javascript/next.js/v14/03-pages/01-building-your-application/09-deploying)と[Upgrading](/ja/javascript/next.js/v14/03-pages/01-building-your-application/10-upgrading)のセクションをチェックアウトしてください。

</PagesOnly>
