Sourceマップは、developmentの間、defaultで有効になっています。productionbuildでは、設定フラグで明示的にオプトインしない限り、sourceがclientに漏れるのを防ぐために無効になっています。

 Next.js は、 production build の間にブラウザの source map 生成を有効にするための設定フラグを提供します。

```js:next.config.js
module.exports = {
  productionBrowserSourceMaps: true,
}
```

`productionBrowserSourceMaps`オプションが有効になると、sourceマップはJavaScriptファイルと同じディレクトリに出力されます。Next.jsはこれらのファイルを要求されたときに自動的に提供します。

- sourceマップを追加すると、`next build`時間が長くなる可能性があります
- I`next build` の間にメモリ使用量が増加します
