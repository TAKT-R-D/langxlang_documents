Source マップは、development の間、default で有効になっています。productionbuild では、設定フラグで明示的にオプトインしない限り、source が client に漏れるのを防ぐために無効になっています。

Next.js は、 production build の間にブラウザの source map 生成を有効にするための設定フラグを提供します。

```js:next.config.js
module.exports = {
  productionBrowserSourceMaps: true,
}
```

`productionBrowserSourceMaps`オプションが有効になると、source マップは JavaScript ファイルと同じディレクトリに出力されます。Next.js はこれらのファイルを要求されたときに自動的に提供します。

- source マップを追加すると、`next build`時間が長くなる可能性があります
- `next build` の間にメモリ使用量が増加します
