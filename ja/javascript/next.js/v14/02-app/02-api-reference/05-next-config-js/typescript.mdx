Next.js はプロジェクトに TypeScript のエラーがある場合、**production build**（`next build`）で失敗します。

アプリケーションにエラーがある場合でも、Next.js による本番コードの生成を強行したい場合は、組み込みの型チェックステップを無効にすることができます。

無効にした場合、ビルドまたはデプロイプロセスの一部として型チェックを実行していることを確認してください。そうでないと非常に危険です。

`next.config.js`を開いて、`typescript`設定で`ignoreBuildErrors`オプションを有効にします：

```js:next.config.js
module.exports = {
  typescript: {
    // !! WARN !!
    // Dangerously allow production builds to successfully complete even if
    // your project has type errors.
    // !! WARN !!
    ignoreBuildErrors: true,
  },
}
```
