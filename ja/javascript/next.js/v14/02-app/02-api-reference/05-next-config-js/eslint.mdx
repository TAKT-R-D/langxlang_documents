あなたのプロジェクトでESLintが検出された場合、errorが存在するとNext.jsはあなたの**production build**(`next build`)に失敗します。

あなたのアプリケーションが ESLint errorを持っていても Next.js が production code を生成するようにしたい場合は、組み込みのリンティングステップを完全に無効にすることができます。これは、既に ESLintをワークフローの別の部分(例えば、CIやpre-commit hook など)で実行するように設定している場合を除き、推奨されません。

`next.config.js`を開き、`eslint` configの中で`ignoreDuringBuilds`オプションを有効にしてください。

```js:next.config.js
module.exports = {
  eslint: {
    // Warning: This allows production builds to successfully complete even if
    // your project has ESLint errors.
    ignoreDuringBuilds: true,
  },
}
```
